# 다언어 대응 방침 (i18n)

**작성일**: 2026-02-10  
**상태**: 방침 확정, 구현 미착수  
**배치 예정**: docs/30_technical/i18n.md (구현 시점에 정식 설계 문서로 승격)

## 1. 대응 언어 및 우선순위

| 우선순위 | 언어 | 시점 | 비고 |
|---------|------|------|------|
| 1 | 日本語 (ja) | Beta 필수 | 1차 테스터 대상 |
| 2 | English (en) | Beta | 기본 프롬프트 언어 |
| 3 | 한국어 (ko) | Beta | 개발 언어 |
| 4 | 中文 (zh) | Release | |
| 5 | Español (es) | Release | |

## 2. 텍스트 분류

| 분류 | 대응 방법 | 예시 |
|------|----------|------|
| A. 정적 텍스트 | i18n key-value (JSON) | UI, 시스템 메시지, 아이템명 |
| B. 반정적 텍스트 | 동일 | 태그명, 판정 결과, fallback 서술 |
| C. LLM 동적 서술 | 프롬프트 언어 지시 전환 | look/move 서술, NPC 대사 |

## 3. 아키텍처 방침

- `src/i18n/` 모듈: 로더 + locale JSON
- 플레이어 설정에 `locale` 필드 추가
- 프롬프트는 영어 단일 작성 + 응답 언어 지시 전환 (품질 차이 체감 시 언어별 분리)
- 데이터 파일 (seed_items.json 등): 설계 시점부터 `name_kr`, `name_ja`, `name_en` 구조

## 4. 로드맵

| 시점 | 작업 | 비고 |
|------|------|------|
| Phase 2 후반 (모듈 구현 중) | i18n 인프라 + `t("key")` 패턴 도입, ko.json 초기 키 | 하드코딩 방지가 핵심 |
| Phase 3 (콘텐츠 확장) | ja.json 번역, LLM 프롬프트 언어 전환 | 노부 직접 번역 |
| Beta | 일본어 QA, 프롬프트 튜닝 | 실 플레이 피드백 기반 |

## 5. 즉시 적용 사항

Gemini에게 데이터 파일 작성 의뢰 시 다언어 필드를 포함시킬 것:
- seed_items.json: `name_kr`, `name_ja`, `name_en`
- 기타 텍스트 데이터도 동일 패턴
